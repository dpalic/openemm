Balises AGNITAS - Aperçu

Le système OpenEMM vous permet d'intégrer les balises de personnalisation spéciales AGNITAS suivantes. Il suffit simplement de les intégrer dans un modèle ou dans un module de texte et de les replacer avant le mailing. Cela vous permet de personnaliser chaque mailing en utilisant les informations contenues dans votre base de données. Les balises vous permettent de vous adresser personnellement à un destinataire ou de personnaliser une partie ou l’entièreté de votre mailing selon les champs auxquels les destinataires accordent un intérêt particulier. Notez cependant que, contrairement aux balises normales HTML, les balises AGNITAS de personnalisation ne doivent pas être fermées par une balise de fin : il n'y a pas de système de fermeture comme pour les balises de contenu dynamique.

  • Balises AGNITAS
  • [agnTITLE type=x]: Permet d’insérer un titre complet dans la lettre d’information, où x est remplacé par 1 ou 2 afin de générer une des formules de politesse suivantes :
    • [agnTITLE type=1] donne : Chère/Cher Madame / Monsieur Nom
    • [agnTITLE type=2] donne : Chère/Cher Madame / Monsieur Nom
    Remarque: Vous pouvez évidemment définir des formules de politesse personnalisées. Il vous suffit de remplacer x par le numéro d’identification de la formule de politesse que vous aurez choisie : [agntitle type=164] donne : la forme de politesse caractérisée par le numéro d’identification 164.
  • [agnFIRSTNAME]: insère le prénom du destinataire.
  • [agnLASTNAME]: insère le nom du destinataire.
  • [agnEMAIL]: insère l’adresse électronique du destinataire.
  • [agnSUBSCRIBERCOUNT]: permet d'afficher le nombre de destinataires du mailing
  • [agnDATE type=x]: insère la date du jour, où x représente le format :
    • [agnDATE type=0]: 0 = exemple: "3.4.2001"
    • [agnDATE type=1]: 1 = exemple: 04/03/2001
    • [agnDATE type=2]: 2 = exemple: "Mardi,le 3 avril 2001"
    • [agnDATE type=3]: 3 = exemple: "2001-04-03"
  • [agnDB column="..."]: insère le contenu de l'un des champs de la base de données de profil. La colonne de paramètre = définit le nom du champ. À titre d'exemple (où le champ SHOESIZE doit être évidemment défini) : [agnDB column="SHOE_SIZE"]
  • [agnIMAGE name="..."]: insère l'image de ce nom à partir des composants graphiques du mailing. Remarque : n’insérez pas le nom de fichier de l’image ici, mais bien le nom attribué à l’image lors de son enregistrement dans les composantes grahpiques.
