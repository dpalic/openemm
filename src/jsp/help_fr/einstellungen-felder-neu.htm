Paramètres - Nouveau champ de profil

OpenEMM ne limite aucunement les entrées parce que chaque base de données de profil d’utilisateur a des exigences différentes en matière d’applications et d’enregistrements. Les Profils peuvent avoir un nombre illimité de champs. Différents types de données sont disponibles. Les dates, une date de naissance, par exemple, sont stockées différemment d’une ville ou d’un nom de rue. Les changements apportés à la structure de la base de données s’appliqueront toujours à tous les destinataires enregistrés dans votre base de données. Vous devez donc décider en temps utile des champs dont vous avez besoin. L’ajout de champs à la base de données peut se faire à n’importe quel moment. Après avoir effectué vos modifications, cliquez sur le bouton Enregistrer pour enregistrer vos paramètres. Si vous voulez revenir en arrière sans enregistrer les changements, cliquez sur le bouton Annuler. Les champs signifient ce qui suit :

  • Nom du champ: nom du champ, qui apparaît dans le menu Options. Par exemple, dans les définitions des groupes cibles.
  • Description: cette entrée définit l'utilisation du champ. Elle est utile si différents utilisateurs travaillent avec OpenEMM.
  • Nom du champ dans la base de données (BD): ce nom apparaît seulement lors de la gestion des champs. Ce champ spécifie le nom qui sera enregistré dans la base de données. Votre entrée sera enregistrée en lettres majuscules, l’underscore (_) servira de séparateur. Par exemple '''CAR'_'USAGE'
  • Type:un champ peut contenir différents types de données. Les types de champs suivants sont disponibles :
    • Date: les champs de ce type enregistrent une date.
    • Numérique: ce champ permet l’entrée de nombres. Mais vous pouvez également définir si l'utilisateur est intéressé (1) ou si il ne l'est pas (0). L’utilisation d’une action vous permet également de compter un champ numérique (par exemple +1 pour chaque réaction de l'utilisateur).
    • Alphanumérique: aucune restriction ne s’applique lorsque vous choisissez ce type de champ. N’utilisez ce type que si vous souhaitez que l'utilisateur puisse saisir toutes les formes possibles d’entrées.
  • Longueur: ce champ définit le nombre maximum de caractères qui peut saisi dans ce champ. Cela ne concerne que les champs alphanumériques. Les champs numériques sont limités à 22 caractères. Il faudrait toujours limiter le nombre de caractères d’un champ alphanumérique à 22.
  • Valeur par défaut:cette zone de texte définit la valeur utilisée depuis le début. Dans les champs numériques, il est recommandé d'utiliser une valeur par défaut de « 0 ». Les champs alphanumériques peuvent aussi être définis par une valeur par défaut 'nulle', par exemple s'il s'agit de la valeur par défaut de la dernière destination de vacances. La date n'a pas besoin de valeur par défaut.

Remarque : si vous avez créé un champ, vous ne pouvez changer que le nom de ce champ et sa description. Tous les autres paramètres ne peuvent plus être modifiés par la suite.
