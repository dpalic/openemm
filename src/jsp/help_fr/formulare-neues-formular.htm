Formulaires – Nouveaux formulaires

Cette fenêtre vous donne la possibilité de créer un nouveau formulaire. Les formulaires offrent une interface client flexible dans OpenEMM. Les clients peuvent s’inscrire eux-mêmes, s’abonner à une liste de diffusion, ou modifier leurs données personnelles – comme, par exemple, le format de courriel désiré - directement sur le site. Remarque : vous devez connaître le codage HTML pour être à même de créer et de gérer des formulaires.

  • Nom: définit le nom des formulaires. Ce nom apparaît plus tard dans OpenEMM lorsque vous avez la possibilité de choisir un formulaire.
  • Description: vous donne la possibilité d'ajouter des commentaires descriptifs au formulaire.

Les étapes préliminaires à la création de formulaires sont maintenant terminées. Allez ensuite dans la section Formulaires et sélectionnez le sous-menu Nouveau formulaire. Commencez par créer un formulaire qui sera utilisé pour afficher des données d’un profil d'utilisateur enregistré. Une définition de formulaire se compose toujours d'une action de départ, d’un formulaire de réussite, d’un formulaire d’erreurs et d’une action de conclusion.

  • Action: dans la liste déroulante Action, située en dessous de la zone Description, sélectionnez l’action que doit effectuer OpenEMM avant d'afficher le formulaire de réussite. Toutes les actions précédemment définies pour cette section s’affichent dans la liste déroulante. Remarque : seules les actions pour lesquelles vous avez spécifié, sous Utilisation, qu’elles devaient être utilisées uniquement pour des formulaires ou des liens et des formulaires apparaissent. Pour notre exemple, sélectionnez Chargement de l’action des données de l'utilisateur que vous avez définie dans la section précédente comme action.
  • Réussite: dans la boîte de dialogue Formulaire de réussite, entrez un texte ou un code HTML pour le site Web où les données du profil de l’utilisateur doivent s’afficher. Vous trouverez plus loin un autre exemple d'affichage du Nom, Prénom, Courriel et Statut de la liste de diffusion.
  • Erreur: la boîte de dialogue Formulaire d’erreurs reprend le texte que le système doit afficher en cas d’erreur. Ici, toutes les commandes HTML sont disponibles comme avant. Aux fins de notre exemple, le texte « Une erreur s'est produite » devrait être suffisant.
  • Action: l’action de conclusion ne sera menée par OpenEMM que dans le cas où l’action de départ a été couronnée de succès et que si l’utilisateur a renvoyé le formulaire de réussite. En d'autres termes, cette action n’est effectuée que lorsque le processus est arrivé à terme. Vous devez d'abord choisir une catégorie.
