Destinataire – Nouveau destinataire

Le système OpenEMM vous permet de créer des destinataires individuellement. Cette fonction n’est intéressante que si vous ne désirez créer que quelques nouveaux destinataires. Dans la plupart des cas, l’importation de données existantes ou l’enregistrement des utilisateurs (qu’ils effectuent eux-mêmes) via le site Web est nettement plus pratique. Cliquez sur Destinataires, puis sur Nouveau destinataire dans la barre de navigation. Une boîte de dialogue apparaît dans la zone de contenu afin que vous entriez les données du nouvel abonné. Celle-ci ressemble à l’écran des nouveaux destinataires et est divisée en deux sections. Les données des nouveaux destinataires doivent être saisies dans les champs de la zone supérieure. Dans la zone suivante, vous introduisez la liste de diffusion à laquelle le destinataire doit être ajouté. Remarque : n’entrez qu’une seule adresse électronique pour le nouveau destinataire. La liste déroulante des formules de salutation ne contient que le sexe du destinataire. La forme exacte de l’adresse est définie ailleurs. Laissez le format du courriel en HTML (défaut) sauf si vous désirez un autre format.

  • Salutation: définissez le sexe du destinataire. La formule exacte de salutation peut être modifiée dans les mailings.
  • PRÉNOM : prénom du destinataire
  • NOM : nom du destinataire
  • E-Mail: affiche l’adresse électronique du destinataire
  • Format du Mail:
    • Texte: Au format texte, tous les clients courriel pourront ouvrir ce message mais les options de formatage (gras, hyperlien, taille de la police, …) ne sont pas accessibles.
    • HTML: Au format HTML, il vous est possible d’ajouter des images et un formatage spécifique au message. Lorsque vous utilisez des images des serveurs externes AGNITAS, vos clients peuvent rencontrer des problèmes avec les interfaces de certains navigateurs, notamment les comptes courriels GMX.
    • HTML Offline: Au format HTML Offline, les images sont en fait intégrées dans le mail comme pièces jointes tandis qu’en version HTML normale, les images sont téléchargées à partir du serveur une fois que le courriel est ouvert.
  • Autres données du profil: la section Autres données du profil contient des champs réservés à une utilisation interne ainsi que des champs définis par l’utilisateur pour les profils des destinataires. Voici les champs réservés à l’utilisation internes :
  • timestamp: date de la dernière modification du profil
  • creation_date: date de la création du profil du destinataire
  • datasource_id: les données ont été importées des champs définis par l’utilisateur. La longueur de cette liste n’est pas limitée et la liste ne se base que sur vos besoins et critères nécessaires : par exemple, les numéros d’identification des clients ou d’autres numéros de référence.

Le sous-menu des Listes de diffusion contient toutes les listes de diffusion créées jusqu’à présent. Le système OpenEMM affiche les listes auxquelles votre destinataire s’est inscrit en les cochant. Si le destinataire s’est inscrit à une liste de diffusion particulière, OpenEMM affiche des données supplémentaires.

  • E-Mail: si cette option est cochée, cela signifie que le destinataire s’est inscrit à cette liste de diffusion. Pour s’inscrire à une liste de diffusion, activez la case à cocher de la liste de diffusion souhaitée.
    Type:
    • Destinataire normal: tous les destinataires qui se sont inscrits à la lettre d’information et qui désirent la recevoir sont classés dans la catégorie des destinataires normaux.
    • Administrateur le créateur de la liste de diffusion s’appelle l’administrateur. Vous devez vous créer un profil de destinataire (si vous ne l’avez pas encore fait) et vous accorder le statut d’administrateur de la liste de diffusion.
    • Destinataires de tests : afin de tester le mailing, il est essentiel d’envoyer un certain nombre de courriels tests pour tester les destinataires. Il ne s’agit pas de véritables destinataires mais d’adresses électroniques auxquelles vous êtes le seul à avoir accès. Vous pouvez également octroyer ce statut à quelqu’un qui contrôlera le mailing avant de l’envoyer à tous les destinataires de la liste.
