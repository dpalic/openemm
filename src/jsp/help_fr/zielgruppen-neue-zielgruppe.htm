Groupe cible – Nouveau groupe cible

Ouvrez l’aperçu des groupes cibles disponibles en cliquant sur le bouton Groupes de la barre de navigation. En cliquant sur le sous-menu Nouveau groupe cible, une boîte de dialogue de saisie de texte s’affiche afin que vous puissiez définir un nouveau groupe cible. Cliquez ensuite sur Enregistrer pour enregistrer votre nouveau groupe cible dans OpenEMM. Pour éditer un nouveau groupe cible, ouvrez le sous-menu Aperçu qui se trouve dans la rubrique Groupes Cibles. Le système affichera alors une liste de groupes cibles disponibles. Cliquez ensuite sur le nom du groupe cible désiré ou sur le symbole Éditer qui se trouve au bout de la ligne. Pour supprimer un groupe cible, cliquez sur le symbole Supprimer.

  • Nom: ans cette zone de texte, vous pouvez introduire un nom significatif pour votre modèle. Ce nom s’affichera plus tard dans les listes d’aperçu et dans les listes déroulantes utilisées pour sélectionner les groupes cibles.
  • Description: Le texte ne doit comporter qu’une ou deux phrases. Une version raccourcie de la description apparaîtra dans l’aperçu et dans la vision détaillée des groupes cibles.
    • Combiner plusieurs règles de recherche:OpenEMM vous permet de combiner plusieurs règles de recherche. Les règles de recherche sont liées par les opérateurs AND et OR. Si, par exemple, un des champs supplémentaires définis dans le profil du destinataire est l’« âge » et que vous désirez afficher la liste de tous les destinataires âgé de 20 à 30 ans possédant une adresse électronique GMX (@gmx.de, @gmx.net, etc.), vous devrez définir trois règles de recherche: E-Mail LIKE %@gmx.% AND AGE_YEARS > 19 AND AGE_YEARS < 31
    • Ouverture des parenthèses: Dans la plupart des cas, une simple règle de recherche (champ, opérateur, valeur de référence) ne sera pas suffisante pour déterminer un groupe cible. Vous pouvez donc définir plusieurs règles de recherche, liées par les opérateurs AND ou OR, comme vous le faites dans une recherche normale. Vous pouvez également utiliser les parenthèses pour les définitions de groupes cibles. L’utilisation de parenthèses vous permet de définir des groupes cibles très complexes. Les règles mathématiques s’appliquent lors de l’utilisation de parenthèses : ce qui se trouve entre parenthèses sera traité en premier. Les parenthèses peuvent également s’emboîter. Toutefois, cela conduit rapidement à une structure de définition cible relativement peu claire. Imaginons que vous ayez défini plusieurs champs de base de données différents pour les intérêts et les hobbies des destinataires (interest_cars, interest_travel c’est-à-dire l’intérêt pour les voitures, pour les voyages, etc.) où 0 signifie pas intéressé et 1 signifie intéressé. Imaginons que vous souhaitez envoyer un courriel contenant des images à deux groupes intéressés uniquement. Vous devez d’abord utiliser le format HTML dans votre courriel. Vous devez ensuite définir les règles de recherche suivantes : (interest_cars = 1 AND interest_travel = 1) AND (type de courriel = 1 OR type de courriel = 2). Le résultat de la recherche contiendra tous les destinataires qui ont coché les cases interest_cars et interest_travel et qui désirent recevoir les courriels au format HTML (1) ou au format HTML Offline (2). Il est plus facile de définir un groupe cible en notant d’abord les règles de recherche comprenant les parenthèses nécessaires sur une feuille de papier.
    • Champs:Les champs internes signifient ce qui suit:
      • BOUNCELOAD: ce compteur mesure le nombre de fois où s’est produit un Soft Bounce.
      • CREATION_DATE: date à laquelle le destinataire a été introduit dans la base de données.
      • CUSTOMER_ID: OpenEMM attribuera automatiquement un numéro d’identification aux nouveaux destinataires. Ce numéro est unique. Chaque destinataire peut être identifié sans erreur possible grâce à ce numéro d’identification.
      • DATASOURCE_ID: Identification de la source de données à partir de laquelle le destinataire a été importé. Si les données CSV sont importées, OpenEMM attribuera automatiquement un numéro d’identification.
      • E-MAIL: Adresse électronique du destinataire
      • FIRSTNAME: Prénom du destinataire
      • GENDER: La règle de recherche sexe = 1, par exemple, filtre tous les destinataires féminins de la base de données (0 = masculin, 1 = féminin, 2 = inconnu).
      • LASTNAME:Nom du destinataire
      • MAILTYPE: Type de courriel choisi par le destinataire. 0 pour texte, 1 pour HTML.
      • ML.TIMESTAMP:Date de la dernière modification du statut d’attribution à la liste de diffusion (inscrit ou désinscrit).
      • TIMESTAMP: Date de la dernière modification du profil.
    • Les opérateurs de relation:Les opérateurs de relation définissent une relation entre le contenu des champs et la valeur de référence. OpenEMM supporte les opérateurs de relation suivants :
      • =: le contenu et la valeur du champ sont identiques. La valeur de référence peut être exprimée en chiffres ou en lettres. Remarque : lorsque vous utilisez l’opérateur « égal », la valeur de référence et le contenu du champ doivent être totalement identiques. Toutefois, la casse des lettres n’est pas prise en compte : « Axel » est identique à « axel ».
      • <> le contenu du champ et la valeur de référence sont différents. La valeur de référence peut être exprimée en chiffres ou en lettres.
      • > le contenu du champ est supérieur à la valeur de référence. La valeur de référence peut exprimée en chiffres ou en lettres. S’il s’agit de texte, la place dans l’alphabet entre en ligne de compte. Par exemple, « b » est supérieur à « a ».
      • < le contenu du champ est inférieur à la valeur de référence. La valeur de référence peut s’exprimer en chiffres ou en lettres. S’il s’agit de texte, la place dans l’alphabet entre en ligne de compte. Par exemple, « a » est inférieur à « b ».
      • IS: il y a deux modificateurs spéciaux : NUL et NON NUL. Ils ne sont pertinents que dans des opérations de la base de données très spécifiques. La base de données fait la différence entre un champ textuel complètement vide et non utilisé et un champ textuel qui contient la chaîne « vide » (chaîne de caractères de longueur 0 représentée par le signe ''). L’opérateur NON NUL repérera tous les champs contenant une chaîne de caractères même s’il s’agit d’une chaîne « vide ». La règle Nom = NULL permettra, par exemple, de filtrer les destinataires dont le nom est manquant. À l’inverse, Nom = '' permettra de trouver les noms qui existaient originellement mais qui ont été supprimés ou remplacés par la chaîne « vide ». Cela fonctionne également avec les champs numériques. Si un champ contient le chiffre 0, la base de données ne considère pas qu’il s’agit d’une chaîne vide. SHOE_SIZE = 0 permettra de trouver les destinataires dont le champ pointure SHOE_SIZE contient en fait l’entrée 0. Inversement, SHOE_SIZE = NUL permettra de trouver tous les destinataires qui n’ont jamais introduit leur pointure dans ce champ spécifique.
      • LIKE: le contenu des champs fonctionne comme un texte de référence contenant des caractères de remplacement. Si vous entrez « %@t-online.de » comme valeur de référence et que vous sélectionnez LIKE comme opérateur, seuls les destinataires dont l’adresse électronique se termine par T-Online seront repris dans la liste. Par exemple, hans@t-online.de et maurer.krause@t-online.de.
      • NOT LIKE: le contenu des champs ne fonctionne par comme un texte de référence contenant des caractères de remplacement. Si vous sélectionnez NOT LIKE, tous les destinataires dont l’adresse électronique ne se termine pas par T-Online seront affichés.
    • Comparaison de valeurs : définit la valeur qui sera recherchée. Par exemple, le champ interne LASTNAME ; Valeur : Smith. Cette fonction permettra de rechercher un destinataire dans le groupe cible dont le nom est Smith.
    • Format de date: l’entrée doit être rédigée comme suit : AAAAMMJJ. Par exemple, 20080301 pour le 01 mars 2008.
