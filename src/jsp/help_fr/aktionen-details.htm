Action - Détails

Une action est utilisée afin de définir ce qu’il faut faire lorsqu’un destinataire clique sur un lien. Cela peut, par exemple, provoquer un changement dans le profil de l’utilisateur du destinataire en augmentant la valeur dans le champ Nombre de 1. Une action peut également engendrer l’envoi d’un mailing. Les actions peuvent comprendre plusieurs étapes. Dans ce chapitre, nous allons aborder séparément la modification des profils des utilisateurs et l’envoi d’un mailing afin que les procédures soient bien claires. Dans la barre de navigation, cliquez sur le bouton Actions. Cliquez ensuite sur le sous-menu Nouvelle action. Le système va alors afficher une boîte de dialogue dans la zone de contenu.

  • Nom: dans la zone Nom, sélectionnez un nom approprié pour l’action. Ce nom s’affichera à chaque fois qu’OpenEMM vous demandera de choisir une action. Afin de vous faciliter la tâche, ce nom doit être significatif.
  • Description: dans cette zone de texte, vous pouvez donner une description de l’objectif attendu de l’action. Ceci prend tout son sens si plusieurs personnes travaillent avec OpenEMM.
  • Utilisation: la liste déroulante a pour objectif de définir la manière dont l’action doit être utilisée. Si vous définissez l’action comme étant « liens uniquement », vous vous assurez que l’action en question n’apparaîtra dans les listes déroulantes que lorsque qu’elle sera liée à un lien. Ceci réduit le nombre d’entrées dans les listes déroulantes. Si vous sélectionnez l’entrée « Uniquement pour les formulaires », cette action spécifique n’apparaîtra dans les listes déroulantes que pour les pages de formulaire. Si vous sélectionnez « Liens et formulaires », cette action sera toujours disponible.

Ensuite, sélectionnez l’action qui doit être exécutée en premier. La liste déroulante Type vous propose les choix suivants : Modification du profil, Envoi du mailing, Inscription, Confirmation de l’inscription, Chargement des données du profil et Désinscription. Ce chapitre est consacré aux profils. Vous devez donc préalablement quitter l’option de Modification du profil. Pour terminer, cliquez sur le bouton Enregistrer. Vous pouvez maintenant définir avec exactitude les champs du profil de l’utilisateur qui doivent être modifiés par le système ainsi que la manière dont ils doivent être changés. La liste déroulante du Nom des colonnes contient maintenant tous les champs, internes et définis par l’utilisateur, dans le profil de l’utilisateur.

  • Modification du profil: l’option de modification du profil édite toutes les informations du profil des destinataires. Dans la liste déroulante du nom des colonnes, sélectionnez les changements de profil.
    • FIRSTNAME:prénom du destinataire.
    • LASTNAME:nom du destinataire.
    • GENDER: 0 = masculin ; 1 = féminin ; 2 = inconnu.
    • MAILTYPE: type de courriel du destinataire. 0 = texte ; 1 = HTML ; 2 = HTML Offline.
    • TIMESTAMP: date des dernières modifications dans le profil du destinataire.
    • CREATION_DATE: date à laquelle le destinataire a été créé.
    • DATASOURCE_ID: identification de la source à partir de laquelle les informations concernant le destinataire ont été importées.
  • Opérateur: Dans la liste déroulante suivante, sélectionnez le signe « + » comme opérateur. Entrez 1 comme valeur dans la dernière case. À chaque ouverture du lien, vous augmenterez ainsi de 1 la valeur dans le domaine interest_travel. Maintenant, cliquez sur Enregistrer pour fermer le champ. Vous pouvez insérer du texte et des chiffres.
  • Envoi d’un mailing basé sur une action: cette étape lance l’envoi d’un mailing prédéfini aux destinataires qui ont déjà cliqué sur un lien ou envoyé un formulaire (voir les paramètres d’ « utilisation »):
    • Mailing: définit la liste déroulante Type permettant d’envoyer le mailing basé sur une action. En cliquant sur le bouton Enregistrer, vous envoyez vos entrées au système. La liste déroulante affiche toutes les mailings basés sur une action que vous avez définis jusqu’à présent dans «Mailing - Nouveau mailing». Sélectionnez le mailing correspondant. La liste déroulante ne montre ni les mailings normaux, ni ceux qui ne sont pas marqués comme « À envoyer » ou ceux qui se basent sur une date.
    • DELAY: cette liste déroulante définit le laps de temps entre le début de l'action et l'envoi des courriels. Il existe différents délais allant d'une heure à sept jours. Le délai est utile quand, par exemple, vous faites des offres individuelles aux destinataires. Toute personne normale s'attendrait à ce que cela prenne au moins plusieurs heures.
  • Inscription: cette étape édite les informations d’une liste de destinataires à partir d’une liste de diffusion. La confirmation d’inscription est définie par défaut. La « Colonne clé pour la vérification des doublons et mises à jour » inclut, par défaut, les courriels pour détecter les doublons. L’option « Utilisation de la confirmation d’inscription » doit être sélectionnée si l’utilisateur doit être intégré dans la liste de diffusion par « Confirmation de l’inscription ».
  • Double Inscription Confirmation: le paramètre de confirmation d’inscription permet d’éviter des inscriptions erronées, contenant des erreurs ou même malveillantes. OpenEMM enverra un courriel de confirmation à l’abonné après son inscription. Ce dernier contient un lien sur lequel doit cliquer l’utilisateur pour confirmer. Le système n’enregistrera pas les données dans la base de données, à moins que l’utilisateur n’ait cliqué sur ce lien. Ce n’est qu’après avoir correctement confirmé son inscription que l’utilisateur reçoit des messages de la liste de diffusion à laquelle il ou elle s’est inscrit. Si l’abonné ne répond pas au courriel de confirmation, le profil est bloqué et automatiquement supprimé d’OpenEMM après trois mois.
  • Chargement des données du profil: après avoir cliqué sur le suivi de lien ou après avoir envoyé un formulaire, les données du profil du destinataire seront chargées dans un formulaire. Cela permet par exemple au destinataire de changer lui-même le format du courriel pour passer du format texte au format HTML.
  • Désinscription: le destinataire sera supprimé de liste s'il clique sur le suivi de lien ou s’il envoie un formulaire.

En cliquant sur « Enregistrer », vous enregistrez l’action que vous venez de terminer. En cliquant sur « Supprimer » et en acceptant la requête, l’action sera supprimée du système.
