Formulaires - Détails

Du point de vue du client, la courtoisie est le maître mot du marketing orienté client. Celle-ci débute avec l’inscription à la lettre d’information. OpenEMM vous offre un service personnalisé qui répond immédiatement aux instructions et attentes du client et du prospect quant à l’adhésion ou l’exclusion, et qui leur permet de modifier les données de leur profil. Les formulaires offrent une interface client flexible dans OpenEMM. Les clients peuvent s’inscrire eux-mêmes, s’abonner à une liste de diffusion, ou modifier les données personnelles de leur profil - comme par exemple le format de courriel désiré- directement sur le site.

  • Nom: définit le nom des formulaires. Ce nom apparaît plus tard dans OpenEMM, où vous pourrez choisir un formulaire.
  • Description: vous donne la possibilité d'ajouter des commentaires descriptifs au formulaire.

Définition de formulaire :
Le concept du formulaire suit une structure claire. La création d’un formulaire se compose toujours d'une action de départ, d’un formulaire de réussite, d’un formulaire d’erreurs et d’une action de conclusion. Ne pas confondre les formulaires et les formulaires Web. Dans le cadre d'un formulaire OpenEMM, les balises HTML définissent les formulaires Web. C’est pourquoi, ils sont appelés formulaires HTML dans ce manuel.

  • L’action de départ prépare OpenEMM à retourner le formulaire de réussite généré à l’étape suivante. L'action correspondante est définie dans la section Actions. L'action peut, par exemple, être configurée pour récupérer les entrées de la base de données pour l'utilisateur à partir de la base de données de profils suite à une modification
  • L'utilisateur verra le formulaire de réussite une fois l’action de départ terminée et réussie. Le formulaire est une page Web qui s’affiche dans le navigateur de l’utilisateur. Son contenu est spécifié en utilisant les balises HTML et des commandes de script.
  • Si l’action de départ se solde par un échec - par exemple, parce que l'utilisateur ne se trouve pas dans la base de données -, le formulaire d’erreurs s’affichera dans la fenêtre du navigateur de l’utilisateur. Pour configurer un formulaire d’erreurs, vous disposez principalement des mêmes possibilités que pour la création d’un formulaire de réussite mais il est globalement impossible de générer un formulaire personnalisé. Pour ce faire, le destinataire devrait préalablement se trouver dans la base de données. Après le retour du formulaire d’erreurs, le processus est terminé et l’action de conclusion ne sera pas effectuée.

L’action de conclusion ne sera menée par OpenEMM que si l’action de départ a été couronnée de succès et que l'utilisateur a renvoyé le formulaire de réussite. En d'autres termes, cette action n’est effectuée qu’après achèvement du processus.
